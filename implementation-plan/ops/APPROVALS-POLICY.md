# Governance and Approvals Policy

## Mandatory board approvals

The following actions require board approval before execution:

1. Pricing model or plan changes
2. Production customer data deletion
3. Authentication/provider credential rotation
4. Secrets provider or storage policy changes
5. Budget overrides above monthly cap
6. Agent termination of core roles (CEO/CTO/CMO)

## Approval payload checklist

Every approval request must include:

- Context and objective
- Impacted systems
- Risk level
- Rollback plan
- Success criteria
- Owner and SLA

## Response SLAs

- Critical requests: <4 hours
- High: <12 hours
- Normal: <24 hours

## Audit requirements

- All approved/rejected actions must be captured in activity stream
- Rejections require reason and alternative path
